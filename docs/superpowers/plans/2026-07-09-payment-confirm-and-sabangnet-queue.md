# Payment Confirm And Sabangnet Queue Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add the first post-order commerce flow: Toss payment confirmation and Sabangnet order submission queue creation.

**Architecture:** Keep the external API call itself out of this slice. The confirm endpoint validates internal order amount, records the Toss payment result, marks the order paid, and creates an idempotent Sabangnet submission record for a later worker.

**Tech Stack:** Django 5, Django REST Framework, pytest-django, SQLite local database.
