# 기획자 확인 요청 체크리스트

작성일: 2026-06-15

## 1. 확인 범위

아래 항목은 쇼핑몰 구축 범위, 외부 연동, 콘텐츠 준비, 운영 정책을 확정하기 위해 필요한 확인사항이다. 각 항목의 담당자, 확보 일정, 제공 자료를 함께 확인한다.

## 2. 회신 방식

각 항목은 아래 형식으로 회신받으면 바로 작업에 반영할 수 있다.

| 구분 | 회신 내용 |
| --- | --- |
| 담당자 | 해당 항목을 확인하거나 자료를 전달할 담당자 |
| 확정값 | 최종 선택값 또는 정책 내용 |
| 제공 자료 | 파일, 링크, 계정 정보, API 문서, 이미지, 영상 URL 등 |
| 확보 예정일 | 자료 전달 또는 계약 완료 예정일 |
| 비고 | 아직 미정인 사유, 업체 답변 대기 여부, 추가 확인 필요사항 |

## 3. 바로 확인할 항목

아래 항목은 먼저 받아야 한다.

| 항목 | 기획자가 해야 할 일 | 필요한 산출물 |
| --- | --- | --- |
| 외부 업체 | 사방넷, PG, 본인인증, 알림톡/SMS 계약 상태 확인 | 업체명, 담당자 연락처, API 문서, 테스트 계정, 사용 가능 예정일 |
| 사방넷 | 가능한 범위에서 최대 연동을 전제로 자료 확보 | API 문서, 테스트 계정, 연동 가능 항목 목록 |
| PG | 결제 방식과 결제수단 확정 | PG 업체명, 지원 결제수단, 지원 연동 방식, 부분 취소 가능 여부 |
| 브랜드 메뉴 | 초기 노출 브랜드 목록과 순서 확인 | 브랜드명, 노출 순서, 대표 배너, 연결 화면 |
| 상품 자료 | 샘플 상품 데이터 준비 | 상품 샘플 20개 이상, 이미지, 옵션, 가격, 재고 |
| 주문/배송 정책 | 운영 정책 확정 | 주문/취소/배송/교환/반품/환불 정책 문구 |
| 홈 영상 배너 | 외부 영상 URL 중심으로 자료 준비 | 영상 URL, 포스터 이미지, 노출 순서, 연결 대상 |
| 1:1 상담 | 외부 솔루션 또는 내부 문의 기능 중 선택 | 선택안, 플로팅 버튼 노출 여부, 상담 운영 방식 |

## 4. 핵심 결정사항

아래 내용은 전체 범위에 영향을 주므로 우선 확인한다.

- 초기 오픈 시 노출할 브랜드 목록과 노출 순서
- 초기 오픈 시 사용할 상품군 목록
- 계약할 외부 업체: 사방넷, PG, 본인인증, 알림톡/SMS
- 외부 연동은 가능한 범위에서 최대한 진행할 예정이므로, 각 업체의 API 문서와 테스트 계정 확보가 최우선
- 1:1 상담 방식: 외부 상담 솔루션 계약 또는 내부 문의 기능 구현 중 선택
- PG 결제 방식은 결제 위젯, API 직접 연동, 리다이렉트 방식 중 업체 지원 범위와 계약 조건에 따라 결정
- 주문/배송/취소/교환/반품 상태와 운영 정책
- 홈 배너 영상 제공 방식: 외부 영상 URL 권장
- 모바일 우선 개발 확정, PC는 모바일 구조를 기준으로 반응형 대응

## 5. 외부 업체 자료 요청

### 사방넷

기획자가 확보해야 할 내용:

- 계약 진행 상태와 사용 가능 예정일
- API 문서
- 테스트 계정
- API에서 제공되는 연동 범위: 상품, 옵션, 재고, 주문, 송장, 취소/반품
- 담당자 연락처

확인이 필요한 이유:

- 상품/옵션/재고 모델 구조가 달라진다.
- 주문 생성 후 사방넷으로 넘기는 시점이 달라진다.
- 송장번호와 배송 상태를 어디서 입력하는지가 달라진다.
- API 문서가 늦게 오면 연동 개발과 테스트 일정이 바로 밀린다.

### PG

기획자가 확정해야 할 내용:

- 계약할 PG 업체
- 사용할 결제수단: 카드, 계좌이체, 가상계좌, 휴대폰 결제, 간편결제
- 업체가 지원하는 연동 방식: 결제 위젯, API 직접 연동, 리다이렉트 방식
- 부분 취소 허용 여부
- 비회원 결제 허용 여부
- 현금영수증/에스크로 필요 여부

결제 연동 방식 설명:

- 결제 위젯: PG사가 제공하는 결제 화면을 쇼핑몰 안에 붙이는 방식이다. 구현과 유지보수가 비교적 단순하고, 카드/간편결제 등 여러 결제수단을 한 화면에서 제공하기 쉽다.
- API 직접 연동: 쇼핑몰이 결제 요청 화면과 흐름을 더 많이 제어하는 방식이다. 커스터마이징 폭은 넓지만 개발과 검증 범위가 커진다.
- 리다이렉트 방식: 사용자가 PG 결제 페이지로 이동했다가 결제 후 쇼핑몰로 돌아오는 방식이다. 모바일 결제에서 자주 쓰이며, 결제 완료/실패 후 복귀 처리를 정확히 설계해야 한다.

확인이 필요한 이유:

- 주문 생성 시점과 결제 승인 시점이 달라진다.
- 결제 실패, 중복 결제, 결제 취소 처리 방식이 달라진다.
- 부분 취소를 허용하면 주문/환불 계산 로직이 더 복잡해진다.

### 본인인증

기획자가 확정해야 할 내용:

- 계약할 본인인증 업체
- 본인인증 사용 위치: 회원가입, 비밀번호 찾기, 주문자 확인
- 본인인증을 필수로 할지 선택으로 할지

확인이 필요한 이유:

- 회원가입 절차와 화면 단계가 달라진다.
- 비밀번호 찾기와 비회원 주문조회 방식이 달라질 수 있다.

### 알림톡/SMS/이메일

기획자가 확정해야 할 내용:

- 사용할 발송 채널: 알림톡, SMS, 이메일
- 발송 대상: 주문 완료, 결제 완료, 배송 시작, 취소/반품 접수, 문의 답변
- 광고성 메시지 발송 여부

확인이 필요한 이유:

- 주문/배송 상태 변경 시 알림 발송 기능이 필요하다.
- 마케팅 수신 동의 항목이 회원가입에 포함될 수 있다.

### 1:1 상담

1:1 상담은 외부 솔루션 계약이 필수는 아니다. MVP에서는 내부 문의 기능으로 구현할 수 있다.

선택지:

- 외부 상담 솔루션: 실시간 채팅, 상담원 배정, 운영 통계가 필요할 때 적합하다. 별도 계약과 스크립트 삽입이 필요하다.
- 내부 문의 기능: 사용자가 문의를 남기고 관리자가 답변하는 게시판형 구조다. 실시간 채팅은 아니지만 초기 개발 범위에는 더 현실적이다.

기획자가 확정해야 할 내용:

- 실시간 채팅이 꼭 필요한지
- 문의 접수/답변 기능만으로 충분한지
- 플로팅 1:1 문의 버튼을 모든 화면에 노출할지

## 6. 브랜드/상품 구조 확인

### 브랜드 메뉴

- 초기 오픈 시 노출할 브랜드 목록
- 브랜드 노출 순서
- 브랜드별 대표 배너 이미지
- 브랜드별 대표 배너 클릭 시 이동할 화면
- 초기 오픈 시 사용할 상품군 목록
- 브랜드/상품군 추가, 숨김, 순서 변경이 운영 중 발생할 수 있다는 전제

기획자가 전달할 형태:

- 브랜드별로 한 줄씩 정리한다.
- 예: `브랜드명 / 노출순서 / 대표배너 파일명 / 연결화면 / 사용 상품군`

### 상품 정보

기획자가 제공해야 할 상품 정보:

- 상품명
- 상품코드
- 브랜드
- 상품군
- 정상가
- 판매가
- 할인 여부
- 대표 이미지
- 상세 이미지 또는 상세 설명
- 색상
- 사이즈
- 재고
- 판매 상태: 판매중, 품절, 숨김

확정이 필요한 정책:

- 품절 상품을 목록에 노출할지
- 신상품 라벨 기준
- 할인 상품 표시 방식
- 상품 상세 설명을 이미지 중심으로 할지, 텍스트도 함께 관리할지

기획자가 전달할 형태:

- 엑셀 또는 스프레드시트로 전달한다.
- 상품 이미지와 상세 이미지는 파일명 또는 URL이 상품코드와 연결되어야 한다.

## 7. 홈/콘텐츠 운영 확인

### 홈 배너

- 홈 배너는 이미지와 영상을 모두 사용할 수 있어야 한다.
- 영상 배너는 외부 영상 URL 사용을 권장한다.
- 영상 사용 시 포스터 이미지를 함께 제공해야 한다.
- 영상 자동재생 여부, 음소거 여부, 반복재생 여부를 확정해야 한다.
- 배너 클릭 시 이동 대상이 필요하다: 상품 목록, 상품 상세, 기획전, 컬렉션, 룩북, 외부 URL.

기획자가 제공해야 할 자료:

- 모바일용 배너 이미지
- PC용 배너가 별도로 필요한 경우 PC용 이미지
- 영상 URL
- 영상 포스터 이미지
- 배너 제목
- 배너 노출 순서
- 배너 연결 대상
- 노출 시작일/종료일

기획자가 전달할 형태:

- 배너별로 이미지/영상 URL/포스터 이미지/링크/노출 순서를 한 줄씩 정리한다.
- 영상은 파일 업로드보다 외부 영상 URL을 우선으로 준비한다.

### 컬렉션/기획전/룩북

확정해야 할 내용:

- 컬렉션과 기획전의 구분
- 각 콘텐츠의 제목, 설명, 대표 이미지
- 콘텐츠 상세에 들어갈 이미지/영상/텍스트
- 콘텐츠에 연결할 상품 목록
- 공개 시작일/종료일
- 노출 순서

기획자가 전달할 형태:

- 콘텐츠별 제목, 설명, 대표 이미지, 상세 이미지/영상, 연결 상품코드를 함께 전달한다.

## 8. 주문/배송/교환/반품 정책

### 주문

- 비회원 주문 허용 여부
- 비회원 주문조회 방식
- 주문 취소 가능 시점
- 부분 취소 허용 여부
- 쿠폰/적립금 사용 가능 여부

### 배송

- 기본 배송사
- 기본 배송비
- 무료배송 기준
- 제주/도서산간 추가 배송비
- 묶음배송 가능 여부
- 송장번호 입력 주체

### 교환/반품/환불

- 교환/반품 신청 가능 기간
- 단순 변심 배송비
- 불량/오배송 처리 기준
- 부분 반품 허용 여부
- 환불 승인 절차
- 반품 불가 상품 기준

기획자가 전달할 형태:

- 고객에게 노출할 정책 문구 기준으로 전달한다.
- 내부 처리 규칙과 고객 안내 문구가 다르면 둘 다 구분해서 전달한다.

## 9. 회원/혜택 정책

### 회원가입/로그인

- 일반 로그인 방식: 아이디 또는 이메일
- 카카오/네이버 소셜 로그인 사용
- 소셜 로그인 후 추가 입력받을 정보
- 회원가입 필수 입력 항목
- 본인인증 필수 여부
- 비회원 구매 허용 여부

### 쿠폰

- 가입 쿠폰 제공 여부
- 쿠폰 종류: 정액, 정률, 무료배송
- 최소 주문금액
- 최대 할인금액
- 브랜드/상품군별 사용 제한
- 중복 사용 가능 여부

### 적립금/등급

- 구매 적립금 지급 여부
- 리뷰 적립금 지급 여부
- 적립금 사용 최소 금액
- 취소/반품 시 적립금 회수 방식
- 회원 등급 운영 여부
- 등급별 혜택

기획자가 전달할 형태:

- 회원가입 필수 입력 항목, 쿠폰 조건, 적립금 조건을 표로 정리한다.
- 쿠폰/적립금/등급을 운영하지 않을 항목은 `미운영`으로 표시한다.

## 10. 검색/필터/정렬 정책

### 검색

- 추천 검색어를 운영자가 직접 등록할지
- 인기 검색어를 노출할지
- 최근 검색어를 저장할지

### 필터

- 사용할 필터 항목: 브랜드, 상품군, 사이즈, 색상, 가격, 배송, 스타일
- 브랜드별 필터 항목이 다른지
- 품절 상품 포함 여부

### 정렬

- 기본 정렬 기준
- 신상품 기준
- 인기상품 기준: 판매량, 조회수, 찜 수 중 선택
- 리뷰순 기준: 리뷰 수 또는 평점

기획자가 전달할 형태:

- 검색어 운영 여부, 필터 항목, 기본 정렬 기준을 표로 정리한다.
- 브랜드별로 필터가 다르면 브랜드별 필터 항목을 따로 표시한다.

## 11. 관리자에서 필요한 운영 기능

기획자가 운영할 필요가 있는 기능만 확인한다.

- 브랜드 추가/수정/숨김/순서 변경
- 상품군 추가/수정/숨김/순서 변경
- 상품 등록/수정/숨김 처리
- 상품 옵션/재고 관리
- 홈 이미지/영상 배너 관리
- 컬렉션/기획전/룩북 관리
- 콘텐츠와 상품 연결 관리
- 주문 조회
- 배송 상태 변경
- 송장번호 입력
- 취소/교환/반품 처리
- 회원 조회
- 쿠폰/적립금 관리
- 리뷰 노출/숨김 처리
- 1:1 문의 답변

기획자가 전달할 형태:

- 운영자가 직접 수정해야 하는 항목과 개발자에게 요청해도 되는 항목을 구분한다.
- 관리자 권한 구분이 필요하면 역할별로 접근 가능한 기능을 표시한다.

## 12. 기획자가 준비해야 할 자료

- 초기 노출 브랜드명과 로고
- 브랜드별 대표 배너
- 초기 상품군 목록
- 상품 샘플 데이터 20개 이상
- 상품 이미지와 상세 이미지
- 홈 배너 이미지
- 홈 영상 URL과 포스터 이미지
- 컬렉션/기획전/룩북 콘텐츠 자료
- 배송/교환/반품 정책 문구
- 이용약관
- 개인정보처리방침
- 사업자 정보
- 고객센터 정보

## 13. 확인 우선순위

### 1순위: 바로 받아야 함

- 외부 업체 계약 대상 확정: 사방넷, PG, 본인인증, 알림톡/SMS
- 각 외부 업체 API 문서와 테스트 계정 확보
- 1:1 상담 방식 확정: 외부 솔루션 또는 내부 문의 기능
- 초기 노출 브랜드 목록과 메뉴 순서
- 초기 상품군 목록
- 주문/배송/취소/교환/반품 정책
- 홈 영상 배너 외부 URL 제공 방식

### 2순위: 화면 작업 전에 받아야 함

- 홈 배너 자료
- 브랜드별 대표 배너
- 컬렉션/기획전/룩북 자료
- 상품 샘플 데이터
- 쿠폰/적립금 정책

### 3순위: 오픈 전에 받아야 함

- 약관/개인정보처리방침
- 사업자 정보
- 고객센터 정보
- 알림톡/SMS 문구
- 리뷰 운영 정책
