# 의류 쇼핑몰 기획 정리본

작성일: 2026-06-15

## 1. 프로젝트 개요

시퀸즈를 메인 브랜드로 하고 여러 의류/수영복 브랜드를 전개하는 편집샵형 온라인 쇼핑몰을 Django 기반으로 제작한다. 단순 상품 판매 사이트가 아니라 홈, 컬렉션, 기획전, 룩북, 리뷰, 검색/필터, 찜, 히스토리, 장바구니, 마이페이지를 포함한 모바일 중심 콘텐츠 커머스 구조다.

원본 자료는 `홈페이지제작(260615)` 폴더의 화면 캡처와 리캡 이미지 기준으로 정리했다. 캡처의 기준 사이트는 `arena.co.kr`이며, 해당 사이트의 구조를 참고하되 시퀸즈 편집샵에 맞게 브랜드명, 카테고리, 상품, 콘텐츠를 교체하는 방향으로 해석한다.

## 2. 목표 일정 및 운영 조건

- 목표 기한: 2026년 8월 말
- 예상 진행: 모든 결정이 순조로우면 2026년 6월 중순 시작, 2026년 7월 말 개발 완료 예상
- 예상 운영비: 최소 월 50달러, 트래픽 증가 시 월 200달러 수준 예상
- 초기 구현 기준: 모바일 우선, PC 화면은 별도 반응형 또는 PC 전용 캡처 기준 필요

## 3. 외부 연동 후보

- 사방넷 API: 상품, 주문, 재고, 배송 등 운영 연동 후보
- PG: 나이스페이, 다날 등 후보
- 본인인증: KG이니시스, KCP, 나이스 등 후보
- 1:1 문의/실시간 문의: 캡처에는 플로팅 버튼 형태로 존재하며 별도 비용 확인 필요

## 4. 전체 정보 구조

전체 구조는 “브랜드를 먼저 고르고, 그 브랜드 안에서 상품군/콘텐츠/세일 상품을 탐색하는 편집샵”으로 잡는다. 시퀸즈 쇼핑몰의 1차 탐색 축은 성별이 아니라 입점 브랜드다.

### 공통 내비게이션

- 상단: 시퀸즈 로고, 검색, 햄버거 메뉴
- 하단 모바일 탭: 홈, 찜, 히스토리, 장바구니, MY
- 플로팅 버튼: 1:1 문의 또는 외부 상담 솔루션 연결
- 푸터: 회사 정보, 약관, 개인정보처리방침, 고객센터, SNS

상단은 전 화면 공통으로 유지한다. 하단 탭은 모바일 핵심 동선이며, 상품 탐색 중에도 홈/찜/히스토리/장바구니/MY로 바로 이동할 수 있어야 한다.

### 1차 IA: 고객이 보는 주요 축

- 홈: 메인 배너, 영상 배너, 추천 상품, 기획전, 컬렉션, 룩북 등 노출
- 브랜드: 입점 브랜드 목록, 브랜드별 대표 배너, 브랜드별 상품군
- 상품: 브랜드/상품군/필터/검색을 통해 접근하는 상품 목록과 상품 상세
- 콘텐츠: 컬렉션, 기획전, 룩북, 리뷰
- 구매: 찜, 히스토리, 장바구니, 주문, 결제, 비회원 주문조회
- 회원: 로그인, 회원가입, 마이페이지, 쿠폰, 적립금, 등급, 주소록
- CS: 1:1 문의, 공지/FAQ, 교환/반품/환불 안내

### 햄버거 메뉴 구조

햄버거 메뉴는 브랜드 중심 탐색 메뉴다. 좌측에 브랜드와 주요 독립 메뉴를 두고, 우측에는 선택된 브랜드의 대표 배너와 하위 상품군을 보여준다.

- 상단: 로그인 / 회원가입
- 좌측 1차 메뉴: 초기 노출 브랜드 목록, SALE, COLLECTIONS, 기획전, 리뷰, 룩북, 장바구니, 마이페이지, 검색
- 우측 브랜드 영역: 선택 브랜드의 대표 이미지 또는 영상 썸네일 배너
- 우측 하위 메뉴: 선택 브랜드의 상품군
- 우측 바로가기: 브랜드 전체보기, 브랜드 기획전, 브랜드 룩북
- 플로팅 영역: 1:1 문의 버튼

브랜드를 선택하면 우측 영역이 즉시 해당 브랜드 기준으로 바뀐다. 상품군을 누르면 `브랜드 + 상품군` 조건이 적용된 상품 목록으로 이동한다. SALE은 독립 메뉴로 두되, 내부에서는 브랜드별 세일 상품을 다시 필터링할 수 있어야 한다. 브랜드와 상품군은 운영 중 추가, 숨김, 순서 변경이 가능해야 한다.

캡처상 `EQUIPMENT`, `이벤트`, `매거진`, `스냅` 등은 일부 제외 표시가 있어 MVP 범위에서는 제외 후보로 본다. 최종 포함 여부는 별도 확정이 필요하다.

### 상품 탐색 흐름

- 홈 배너/영상 배너 -> 기획전 또는 상품 목록
- 햄버거 메뉴 브랜드 선택 -> 브랜드 홈 또는 브랜드 상품 목록
- 브랜드 하위 상품군 선택 -> 브랜드 + 상품군 상품 목록
- 검색어 입력 -> 검색 결과 상품 목록
- 컬렉션/기획전 상세 -> 연결 상품 목록
- 찜/히스토리 -> 상품 상세 또는 장바구니

상품 목록은 모든 진입점에서 공통 컴포넌트를 사용한다. 진입 경로에 따라 기본 필터만 달라진다.

### 콘텐츠 탐색 흐름

- 컬렉션: 여러 기획성 배너를 모아 보여주고, 상세에서 연결 상품을 노출
- 기획전: 특정 프로모션/시즌 캠페인 상세와 관련 상품을 노출
- 룩북: 시즌별 브랜드 비주얼 콘텐츠를 이미지 중심으로 노출
- 리뷰: 구매자 후기와 포토 리뷰를 모아 노출

콘텐츠는 상품 판매로 이어져야 한다. 컬렉션/기획전은 상품 연결이 필수이고, 룩북은 브랜드 이미지 중심이지만 필요 시 관련 상품을 연결할 수 있게 둔다.

### 구매 및 회원 흐름

- 비회원: 상품 탐색, 검색, 장바구니, 비회원 주문조회 가능 범위 확정 필요
- 회원: 찜, 히스토리, 쿠폰, 적립금, 등급, 주문내역, 주소록 사용
- 장바구니: 옵션/수량 변경, 선택 삭제, 관심상품 이동, 주문하기
- 주문/결제: PG 결제, 배송지 입력, 쿠폰/적립금 사용, 주문 완료
- 마이페이지: 주문 상태, 취소/교환/반품, 쿠폰/적립금, 회원 정보 관리

### 관리자 정보구조

- 브랜드 관리: 브랜드명, 로고, 대표 배너, 노출 순서, 노출 여부
- 상품 관리: 상품, 옵션, 재고, 가격, 이미지, 필터 속성
- 콘텐츠 관리: 홈 배너, 영상 배너, 컬렉션, 기획전, 룩북, 리뷰 이벤트
- 주문 관리: 주문, 결제, 배송, 취소, 교환, 반품
- 회원 관리: 회원, 등급, 쿠폰, 적립금, 주소록
- CS 관리: 1:1 문의, FAQ, 공지, 약관
- 연동 관리: 사방넷, PG, 본인인증, 알림 발송 로그

## 5. 카테고리 구조

시퀸즈 편집샵의 기본 탐색 구조는 브랜드별 분류다. 브랜드를 1차 축으로 두고, 각 브랜드 안에서 성별/상품군/스타일/세일 여부를 하위 분류 또는 필터로 제공한다. 브랜드와 상품군은 고정값이 아니라 운영 중 늘거나 줄 수 있는 관리 대상이다.

### 1차 분류: 브랜드

- 초기 노출 브랜드 목록
- 브랜드 노출 순서
- 브랜드 노출/숨김 상태

초기에는 참고 화면 기준으로 3개 내외의 브랜드가 노출될 수 있으나, 구조는 브랜드 수가 변해도 대응 가능해야 한다.

### 상품군 예시

- 상의
- 하의
- 아우터
- 가방
- 소품
- 원피스
- 비키니
- 래쉬가드 / 커버업
- 워터 레깅스 / 팬츠
- 세트

### 브랜드별 보조 분류

- 성별: 여성, 남성, 키즈, 공용
- 라인/스타일: 퍼포먼스, 레저, 어패럴, 레이싱, 프로페셔널, 엘리트, 트레이닝, 아쿠아
- 판매 상태: 신상품, 세일, 품절 제외
- 콘텐츠 연결: 브랜드별 기획전, 컬렉션, 룩북

따라서 Django 데이터 모델은 `Brand`를 독립 엔티티로 두고, `Product`가 브랜드와 필수 연결되도록 설계한다. `Category`는 전역 카테고리로 둘 수도 있고 브랜드별 카테고리 트리로 둘 수도 있으나, 햄버거 메뉴 노출은 브랜드를 기준으로 조립되어야 한다.

## 6. 주요 화면 요구사항

### 홈

홈 화면은 브랜드 캠페인과 상품/콘텐츠 섹션을 세로로 배치한다. 홈 배너는 이미지뿐 아니라 영상을 재생할 수 있어야 한다.

- 메인 비주얼 배너: 이미지 또는 영상
- 영상 배너: MP4 등 웹 재생 가능한 포맷, 모바일 자동재생 정책 고려
- 프로모션/기획전 배너
- Trending Now 또는 주요 추천 섹션
- 룩북 섹션
- 매거진 또는 콘텐츠 섹션
- Style Picks
- SNAP 또는 사용자/브랜드 스타일 콘텐츠
- 하단 푸터 및 고객센터 정보

캡처상 매거진, SNAP 일부 영역에는 제외 표시가 있으므로 MVP에서는 홈 섹션을 줄일 수 있다.

### 상품 목록

상품 목록은 카테고리, 기획전, 컬렉션, 검색 결과에서 공통으로 사용한다.

- 2열 상품 그리드
- 상품 이미지
- 신상품/할인 등 라벨
- 상품명
- 정상가/판매가/할인가
- 찜 버튼
- 장바구니 빠른 추가 버튼
- 페이지네이션
- 정렬
- 필터

### 필터

캡처에서 확인된 필터 항목은 다음과 같다.

- 사이즈: M, L, 070, 075, 080, XS(80), S(85), M(90), L(95), XL(100), 2XL 등
- 컬러: 색상 스와치 기반
- 종류: 반신, 원피스 등
- 배송: 일반배송
- 가격: 구간별 필터
- 상품 속성: 백스타일 등

필터는 상품 속성 테이블로 관리해 카테고리별로 다른 필터를 노출할 수 있게 구성한다.

### 정렬

정렬 모달에서 확인된 항목은 다음과 같다.

- 신상품
- 상품명
- 낮은가격
- 높은가격
- 제조사
- 인기상품
- 사용후기
- 조회수

### 검색

검색 화면은 추천 검색어, 최근 검색어, 인기 검색어를 제공한다.

- 검색 입력창 placeholder: 나이대, 색상, 가격대, 스타일
- 추천 검색어 예시: 미스티레이, 래쉬가드, 수모
- 최근 검색어
- 인기 검색어 및 집계 시각

검색 대상은 상품명, 상품코드, 브랜드, 카테고리, 색상, 스타일 태그를 포함하는 것이 적절하다.

### 로그인 / 회원가입

로그인 화면은 자체 로그인과 카카오/네이버 소셜 로그인을 함께 제공한다. 애플 로그인은 사용하지 않는다.

- 카카오 로그인 / 회원가입
- 네이버 로그인 / 회원가입
- 아이디 로그인
- 비밀번호 입력 및 표시/숨김
- 아이디 찾기
- 비밀번호 찾기
- 회원가입
- 선등록
- 비회원 주문조회

회원가입 혜택 예시는 다음과 같다.

- 1만원 쿠폰
- 세일 상품 구매
- 매월 10% 쿠폰
- 리뷰 적립금
- 등급별 혜택

### 컬렉션

컬렉션은 배너 목록과 연결 상세 페이지로 구성한다.

- 컬렉션 배너 목록
- 각 배너 클릭 시 상세 페이지 이동
- 상세 페이지에는 제목, 설명, 연결 상품 목록 노출
- 예시 컬렉션: 그래픽데이, 솔라피에스타, 솔라랩 & 아이스핏 등

### 기획전

기획전은 메인 배너, 기획전 목록, 기획전 상세, 관련 상품 목록으로 구성한다.

- 기획전 목록 카드
- 기획전 상세 제목 및 이미지/영상 콘텐츠
- 관련 상품 목록
- 상세 내 상품 바로 구매 또는 장바구니

### 룩북

룩북은 시즌별 브랜드 비주얼 콘텐츠다.

- 룩북 목록
- 시즌 표기
- 제목
- 요약 설명
- 대표 이미지
- 상세 설명
- 이미지 갤러리
- 목록 복귀 버튼

### 리뷰

리뷰 화면은 리뷰 이벤트 안내와 리뷰 목록을 포함한다.

- 리뷰 이벤트 배너
- 리뷰 작성 혜택 안내
- 이미지 포함 리뷰 목록

- 평점 또는 별점
- 리뷰 작성자 정보
- 리뷰 이미지
- 리뷰 상세/더보기

### 찜

찜 화면은 회원의 위시리스트 상품을 관리한다.

- 찜한 상품 목록
- 상품 이미지, 라벨, 상품명, 가격
- 상품 삭제
- 장바구니 담기
- 바로 주문
- 페이지네이션

### 히스토리

최근 본 상품 화면은 사용자가 본 상품을 보여준다.

- 최근 본 상품 목록
- 상품 이미지, 라벨, 상품명, 정상가/판매가
- 삭제
- 장바구니 담기
- 바로 주문
- 페이지네이션

- 전체 선택
- 선택 삭제
- 상품별 선택 체크박스
- 상품 이미지
- 상품명
- 옵션
- 수량
- 옵션 변경
- 수량 변경
- 관심상품 이동
- 주문하기
- 총 상품 금액
- 배송비
- 결제 예정 금액
- 이용 안내
- 추천 상품

### 마이페이지

- 회원명
- 멤버십 등급
- 등급 산정 기간
- 등급 만료일
- 구매 금액
- 사용 가능 쿠폰
- 누적 적립금
- 다음 등급까지 필요한 구매 금액
- 주문처리 현황: 입금전, 배송준비중, 배송중, 배송완료
- 취소, 교환, 반품 현황
- 주문내역조회
- 적립금 내역
- 쿠폰 내역
- 배송 주소록 관리
- 최근 본 상품
- 나의 위시리스트
- 나의 게시글
- 회원 정보 수정
- 로그아웃

## 7. Django 앱 구성 초안

### accounts

회원, 카카오/네이버 소셜 로그인, 등급, 쿠폰, 적립금, 주소록, 본인인증을 담당한다.

### catalog

브랜드, 카테고리, 상품, 옵션, 사이즈, 컬러, 재고, 가격, 상품 이미지, 상품 속성을 담당한다.

### commerce

장바구니, 주문, 결제, 배송, 취소, 교환, 반품, 비회원 주문조회를 담당한다.

### content

홈 이미지 배너, 홈 영상 배너, 기획전, 컬렉션, 룩북, 리뷰 이벤트, 매거진/SNAP 포함 여부를 담당한다.

### reviews

상품 리뷰, 이미지 리뷰, 평점, 리뷰 적립금 지급 정책을 담당한다.

### support

1:1 문의, 고객센터, 게시판, FAQ를 담당한다.

### integrations

사방넷, PG, 본인인증, 알림톡/문자/메일 등 외부 연동을 담당한다.

## 8. MVP 범위 제안

2026년 8월 말 공개를 목표로 하면 다음 범위를 1차 MVP로 잡는 것이 현실적이다.

- 회원가입/로그인/소셜 로그인
- 상품 카테고리 및 상품 목록
- 상품 상세
- 검색, 필터, 정렬
- 찜
- 최근 본 상품
- 장바구니
- 주문/결제
- 마이페이지 주문 현황
- 기획전/컬렉션/룩북
- 리뷰 목록 및 리뷰 작성
- 기본 관리자 기능
- 사방넷/PG/본인인증 중 필수 연동

이벤트, 매거진, SNAP, 고급 멤버십 자동화, 실시간 1:1 상담, 스마트 픽업, 오프라인 매장 연동은 2차 범위로 분리할 수 있다.
